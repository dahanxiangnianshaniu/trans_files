# 留存期合规性判定详细规则

阶段四D检查时按需加载。本文件涵盖留存期完整性判定、上限合规比对逻辑、一致性比对维度、⊘标记规则、删除能力判定规则、关键词扫描降级策略。

## 1. 留存期完整性判定（4D-1）

### 核心区分："不被留存" vs "未定义留存期"

| 状态 | 原文表述 | 判定 | 说明 |
|------|---------|------|------|
| **不被留存（合法）** | "不留存""不存储""不保存""不保留""无需留存" | 合法，不计入缺失 | 明确声明不留存，不需要定义留存期 |
| **被留存但未定义留存期（违规）** | 留存期字段为空、空白、未填写 | FAILED | 被留存的数据必须定义留存期 |
| **留存期有定义** | 留存期字段有具体值或策略描述 | PASS | 无论值是否合规（合规性在4D-2/4D-3检查） |

### 判定流程

1. 读取DI清单中每条个人数据的留存期列值
2. 对每条数据判断其留存期状态：
   - 留存期值含"不留存""不存储"等否定词 → 合法（不计入缺失）
   - 留存期值为空或未填写 → 检查上下文（同一行的其他列是否暗示该数据被留存）
   - 留存期有具体值或策略描述 → PASS
3. 统计缺失条目数和不被留存条目数

### 常见误判

| 误判 | 正确做法 |
|------|---------|
| 留存期为"不留存"标FAILED | 标合法——"不留存"是明确声明 |
| 留存期为空一律标FAILED | 需先确认该数据是否被留存——如果上下文暗示不留存，则不违规 |
| 留存期为策略描述（如"注销后删除"）标FAILED | 标PASS——4D-1检查完整性，策略描述算有定义 |

## 2. 留存期上限合规比对逻辑（4D-2）

### 比对流程

1. **数据类型匹配**：将DI清单中的数据类型与留存规范中的数据类型做匹配
2. **数值比较**：将清单中的留存期时长与规范中对应类型的上限时长做比较
3. **判定**：留存期超过上限 → FAILED；不超过 → PASS

### 数据类型匹配规则

- 匹配必须基于 Read 工具读取的真实内容，禁止 LLM 推断或猜测
- DI清单中的数据类型在留存规范中找不到明确对应 → WARNING（不标 FAILED）
- 数据类型名称有差异但明显指向同一类型（如"IP地址" vs "IP地址信息"）→ 可以匹配，但需在报告中说明匹配依据

### 非数值型留存期处理

DI清单中可能存在策略性描述而非数值型留存期：

| 留存期类型 | 原文示例 | 处理方式 |
|-----------|---------|---------|
| 策略描述 | "留存至业务目的达成""注销后删除""永久存储" | WARNING：无法进行数值比对，需人工确认是否合规 |
| 条件触发 | "管理员手动删除""达到转储条件后删除" | WARNING：无法确定具体时长 |
| 设备关联 | "与设备生命周期一致""随设备删除而删除" | WARNING：设备留存期本身不确定 |

### 比对结果判定

| 比对结果 | 判定 | 说明 |
|---------|------|------|
| 留存期数值 ≤ 规范上限 | PASS | 合规 |
| 留存期数值 > 规范上限 | FAILED | 超上限 |
| 数据类型匹配不确定 | WARNING | 避免误判 |
| 留存期为非数值型描述 | WARNING | 无法数值比对 |
| DI清单或留存规范为⊘ | ⊘ | 无法检查 |

## 3. 留存期对外一致性比对维度（4D-3）

### 比对维度

1. **遗漏检查**：DI清单中的数据类型是否在隐私声明中有所覆盖
2. **时长匹配检查**：DI清单中的留存期时长是否与隐私声明中的描述一致

### 比对逻辑

| 比对结果 | 判定 | 说明 |
|---------|------|------|
| DI清单中有数据类型但隐私声明未提及 | FAILED | 遗漏 |
| DI清单留存期与隐私声明描述不一致 | FAILED | 时长不匹配 |
| 类型匹配不确定 | WARNING | 避免误判 |
| 隐私声明存留期限为△模糊描述 | WARNING | 无法精确比对 |
| DI清单留存期为策略描述 | WARNING | 无法精确比对 |

### 隐私声明描述模糊的处理

当隐私声明中对应数据类型的存留期限为△（模糊描述）时：

| 隐私声明描述 | DI清单留存期 | 处理方式 |
|-------------|-------------|---------|
| "短期保存"（模糊） | "30天"（具体） | WARNING：隐私声明△模糊，无法与DI清单精确比对 |
| "根据业务需要"（模糊） | "永久存储"（策略） | WARNING：双方均为非精确描述，需人工确认 |
| "注销后删除"（△策略） | "90天"（具体） | WARNING：隐私声明为策略描述，无法与DI清单精确比对 |
| "保存90天"（✓具体） | "90天"（具体） | PASS：一致 |

### 复用原则

4D-3复用阶段三已提取的隐私声明留存期描述，不重复提取。

## 4. ⊘标记规则

### ⊘触发条件

| 步骤 | ⊘触发条件 |
|------|---------|
| 4D-1 | DI清单未找到（0个）或找到多个 |
| 4D-2 | DI清单为⊘ 或 留存规范未找到（0个）或找到多个 |
| 4D-3 | DI清单为⊘ 或 隐私声明为⊘（多个隐私声明时） |

### ⊘语义

- ⊘ ≠ ✓（不表示通过）
- ⊘ ≠ ✗（不表示失败）
- ⊘ ≠ △（不表示模糊）
- ⊘ = "因文档缺失无法检查"，需人工复核
- ⊘ 计入最终判定时产生 WARNING

### ⊘对阶段四的影响

⊘不影响阶段四的内部判定结果：
- 阶段四✓仍为✓，✗仍为✗，△仍为△
- ⊘仅在最终整体判定时计入，产生WARNING

## 5. 删除能力判定规则（4D-4）

### 关键词扫描流程

**第一轮关键词**：
- 删除个人信息、删除隐私数据、删除个人数据、remove personal data、erase personal information
- 数据销毁、data erasure、数据擦除、clear personal data

**第二轮扩大关键词**（仅第一轮无命中时执行）：
- 清除数据、erase、wipe、重置、reset
- 匿名化、anonymize、脱敏、desensitize

### 判定规则

| 命中结果 | LLM判断 | 最终判定 |
|---------|---------|---------|
| 找到关键词 + 文档描述了删除方法/入口 | ✓ | ✓ |
| 找到关键词 + 仅提及但未描述具体方法 | ⚠️ | ⚠️ |
| 扩大搜索后仍无命中 | ✗ | ✗ |

### 二进制格式处理

xlsx/chm等二进制格式文档，Grep全文搜索可能无法命中关键词：
- 使用 Read 工具读取文档内容后由 LLM 判断是否包含删除能力描述
- 不得仅因Grep无命中就判定✗，需确认已用Read工具替代检查

### 检查范围

本步骤检查"有没有删除个人数据能力描述"，不深入验证机制的具体实现和有效性。

## 6. 关键词扫描降级策略

### MCP可用时

使用 `search_in_document` 工具进行关键词搜索。

### MCP不可用时

1. 优先使用 Grep 工具全文搜索关键词
2. 对二进制格式文件（xlsx/chm等），Grep无法命中时使用 Read 工具读取内容后 LLM 判断
3. 仍无法满足时使用 Bash+Python 脚本读取文档内容后 LLM 判断
4. 在报告中标注降级模式
